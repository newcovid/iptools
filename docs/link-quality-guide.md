# 链路质量评测指南

> 配套工具：诊断（Diagnostics）→ 链路质量（Link Quality）。
> 本文解释每个参数代表什么、数值偏高/偏低意味着什么、评分公式如何得出，
> 以及出现问题时可能的排查方向。读完后你应能凭这一个面板大致判断「当前网络好不好、
> 瓶颈在哪、下一步查什么」。

---

## 1. 它在测什么

链路质量工具从**你选定的那块网卡**发出一连串 ICMP 探测包（可指定目标、包数、间隔、
超时、包大小），在测试全程**持续采样**，最终给出一个 0–100 的综合评分与分项明细。

- **有线网卡**：评估连通性三要素——丢包、延迟、抖动，并实时读取协商链路速率。
- **无线网卡**：在连通性之外，额外采集射频与制式参数（RSSI、信号质量、信道/频段、
  PHY 制式、协商收发速率、BSSID、认证/加密），并纳入评分。

探测包经 `IcmpSendEcho2Ex` **绑定所选网卡的源 IP** 发出，因此多网卡并存时，
测得的就是「这条链路」的真实表现，而不是系统默认路由那条。

## 2. 各参数怎么看

### 2.1 连通性（有线 / 无线通用）

| 参数 | 含义 | 越低/越高代表 | 经验阈值 |
|---|---|---|---|
| **延迟 min/avg/max (ms)** | 往返时延 RTT 的最小/平均/最大值 | 越低越好。`avg` 看整体，`max` 看偶发卡顿 | 局域网 <5ms；到公网网关 <30ms 优秀，>100ms 偏高 |
| **抖动 jitter (ms)** | 相邻两次延迟之差的平均（延迟的稳定度） | 越低越好。抖动大 = 时延不稳，实时音视频/游戏体验差 | <10ms 良好，>30ms 明显 |
| **丢包 loss (%)** | 未收到回包的比例 | 越低越好。丢包是体验杀手，权重最高 | 0% 理想，>1% 需关注，>5% 已影响使用 |
| **已发/已收 sent/recv** | 已发出与已回包的探测计数 | 反映进度与丢包来源 | — |

> **min/avg/max 的用法**：`avg` 高 → 链路整体慢（距离远、绕路、设备转发慢）；
> `avg` 正常但 `max` 突高 → 偶发拥塞或干扰；三者都低且接近 → 链路干净稳定。

### 2.2 无线专属（射频与制式）

| 参数 | 含义 | 越低/越高代表 | 经验阈值 |
|---|---|---|---|
| **RSSI (dBm)** | 接收信号强度，**负值**，越接近 0 越强 | 越高（越接近 0）越好 | ≥-50 极佳；-60 良好；-70 可用；≤-80 很弱 |
| **信号质量 (%)** | 驱动给出的归一化信号质量 | 越高越好 | >70% 良好，<40% 偏弱 |
| **信道 / 频段 / 频率** | 当前工作信道、2.4/5/6GHz、中心频率 | 2.4G 穿墙强但拥挤慢；5/6G 快但衰减快 | 2.4G 优选 1/6/11；5G 看是否 DFS/拥挤 |
| **PHY 制式** | Wi‑Fi 代际（Wi‑Fi 4/5/6/7 等） | 越新越好（更高带宽、更优调制） | Wi‑Fi 6/7 最佳 |
| **Tx/Rx 协商速率 (Mbps)** | 与 AP 协商的发送/接收物理速率 | 越高越好；随距离/干扰动态变化 | 与制式+信道带宽匹配；骤降即信号劣化 |
| **BSSID** | 当前连接 AP 的 MAC | 用于辨别漫游到了哪个 AP | — |
| **认证 / 加密** | 如 WPA3‑Personal / AES | 越新越安全 | 优先 WPA3/WPA2‑AES，避免 WEP/TKIP |

### 2.3 链路速率（有线）

显示的是网卡与对端（交换机/路由器）**协商的物理链路速率**（如 1Gbps）。
它是链路的「容量上限」，不是当前实际吞吐量。

- **它通常恒定**——这是正常现象，不是卡死。本工具在测试期间**每轮实时重读**，
  因此一旦发生**降速重协商**（如网线/接口劣化使 1Gbps 退回 100Mbps）或**链路断开**，
  数值会随之变化或变为「-」，如实反映链路状态。
- 想测「实际能跑多快」请用**内网测速**（iperf 风格对测）或**公网测速**。

## 3. 评分公式

总评分是各维度子评分（0–100）的**加权平均**：

```
总分 = Σ(子评分 × 权重) / Σ权重
```

### 3.1 维度与权重

| 维度 | 有线权重 | 无线权重 |
|---|---|---|
| 丢包 loss | 40 | 25 |
| 延迟 latency | 35 | 20 |
| 抖动 jitter | 25 | 15 |
| 信号 signal (RSSI) | — | 25 |
| 速率 rate (Tx) | — | 10 |
| 制式 phy | — | 5 |

> 有线只看连通性，故丢包/延迟/抖动权重更重；无线把信号质量提到与连通性相当的高度，
> 因为无线的连通性问题大多源于射频。**面板会用 `◀` 高亮当前最弱（权重最低分占比）维度**，
> 直接指出短板。

### 3.2 各维度子评分（线性映射，夹紧到 0–100）

| 子评分 | 满分(100) 对应 | 零分(0) 对应 |
|---|---|---|
| 延迟 | ≤20 ms | ≥300 ms |
| 抖动 | ≤2 ms | ≥80 ms |
| 丢包 | 0 % | ≥10 % |
| 信号(RSSI) | ≥-50 dBm | ≤-85 dBm |
| 速率(Tx) | ≥433 Mbps | ≤6 Mbps |
| 制式 | Wi‑Fi 6/7 = 100 | Wi‑Fi 5 = 80 / Wi‑Fi 4 = 60 / 更旧 = 30 |

中间值线性插值。例如平均延迟 160ms ≈ (300-160)/(300-20) ≈ 50 分。

### 3.3 评级分档

| 总分 | 评级 | 含义 |
|---|---|---|
| ≥85 | 优秀 Excellent | 各项健康，放心使用 |
| ≥70 | 良好 Good | 体验良好，个别项有提升空间 |
| ≥50 | 一般 Fair | 能用但有明显短板，看 `◀` 维度 |
| <50 | 较差 Poor | 体验受影响，需排查 |

> **边界规则**：全程一个回包都没有（recv=0）时，延迟/抖动按最差计分（避免「0ms」被误判为满分），
> 此时丢包 100% 会把总分压到 Poor；完全没发出探测（sent=0）则不给评分（显示「-」）。

## 4. 排查方向速查

| 现象 | 可能原因 | 下一步 |
|---|---|---|
| 丢包高、延迟/抖动正常 | 干扰、半双工冲突、线缆/接口接触不良、AP 过载 | 换线/换口；无线换信道/靠近 AP；看是否高峰期 |
| 延迟 avg 高 | 绕路、上游拥塞、目标远、设备转发慢 | 配合**路由跟踪**看哪一跳变高；换近一点的目标验证 |
| 抖动大但 avg 不高 | 间歇性拥塞、无线干扰、省电模式 | 关网卡省电；无线避开拥挤信道；有线查 QoS/限速 |
| 无线 RSSI 弱（≤-75dBm） | 距离远、隔墙、天线朝向、AP 发射弱 | 靠近 AP；换 5G→2.4G 提升穿墙；加 AP/Mesh |
| 无线速率(Tx)远低于制式上限 | 信号弱触发降速、信道带宽窄、邻频干扰 | 提升 RSSI；检查信道带宽(20/40/80MHz)；避开 DFS |
| 有线速率只有 100Mbps（期望 1G） | 网线只支持百兆/线序坏、网口/交换机限速 | 换 Cat5e+ 网线；换网口；查交换机端口协商 |
| 有线速率显示「-」或测试中变化 | 链路断开/重协商 | 检查物理连接与网卡状态 |
| 制式偏旧（Wi‑Fi 4/5） | 网卡或 AP 不支持新制式 | 升级网卡/AP；确认路由开启了对应频段 |

## 5. 常见疑问

**Q：有线链路速率为什么一直不变？**
A：那是**协商的物理链路速率**（容量上限），稳定是正常的。工具已每轮实时重读，
只有真正降速/断链时才会变。想看「实际吞吐」用内网/公网测速。

**Q：为什么我选的网卡和默认上网的不是同一块，结果不同？**
A：本工具按所选网卡的源 IP 发包，测的就是那条链路。多网卡（如同时插网线+连 Wi‑Fi）时，
分别选中即可对比两条链路的质量。

**Q：评分偏低但我感觉网络没问题？**
A：看分项条与 `◀` 高亮的最弱维度。例如到 `8.8.8.8` 延迟天然偏高会拉低延迟分，
可改测网关或就近节点；评分是相对阈值的客观量化，不等于「不可用」。
