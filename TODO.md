本文档记录已确认的问题、缺陷和未来改进方向。条目仅代表规划，不表示当前版本已经实现。

## 当前基线

- Windows 链路质量诊断已能识别并展示当前连接的 802.11a/n/ac/ax/be PHY 制式，以及
  SSID、BSSID、RSSI、信号质量、频段、信道、频率、Tx/Rx 协商速率和认证/加密信息。
- Linux 当前只能从 `iw dev <interface> link` 获取部分连接信息；PHY 制式、Wi-Fi 代际、
  认证和加密仍会退化为未知值。
- 适配器页面目前仅展示 SSID 和通用网卡信息；完整无线详情只出现在链路质量诊断中。
- 链路质量综合分由 ICMP 丢包、延迟、抖动及无线 RSSI、Tx 速率、PHY 制式组成。

## P0：正确性缺陷

- [ ] **缺失无线遥测不得按低质量计分。**
  - 当前无线查询失败或字段未知时，缺失的 RSSI、速率和 PHY 会分别按最差值或低档值计分，
    无法区分“链路差”和“平台/驱动未提供数据”。
  - 将不可用维度显示为 `N/A`，从有效权重中排除，并明确提示无线遥测缺失及原因。
  - 增加无线快照完全缺失、部分缺失和采样中断的评分测试。

- [ ] **分离本地无线链路质量与公网路径质量。**
  - 默认目标 `8.8.8.8` 的结果同时受无线链路、路由器、运营商、互联网路径和目标 ICMP
    策略影响，不能单独代表 Wi-Fi 质量。
  - 自动识别并测试本地网关，同时保留用户指定的公网目标，分别给出“本地链路”和
    “端到端路径”结果；评分阈值应与目标类型匹配。

- [ ] **持续更新会变化的无线字段。**
  - 当前测试开始时只保存一次 Tx/Rx 速率、BSSID、信道、频段和 PHY；测试循环仅持续更新
    RSSI 与信号质量，可能显示过期速率，也无法发现漫游、切信道和制式回退。
  - 将 Tx/Rx 速率、BSSID、信道、频段和 PHY 纳入时间序列；对漫游、断联、重新关联和
    速率骤降生成事件或醒目标记。

## P1：无线分析核心能力

- [ ] **区分当前制式与支持能力。**
  - 分别展示当前协商 PHY、客户端网卡支持的制式和 AP/BSS 广播的制式能力。
  - 能正确表达 `A/N/AC` 这类兼容模式，而不把它误认为某次连接同时使用多个制式。
  - 覆盖 802.11a/b/g/n/ac/ax/be，并正确标注 Wi-Fi 4/5/6/6E/7。

- [ ] **补齐链路协商参数。**
  - 增加信道宽度、主/中心信道、MCS、NSS/空间流数、Guard Interval 和调制信息。
  - 根据 PHY、频段、信道宽度、MCS 和 NSS 解释当前 Tx/Rx 协商速率是否合理。

- [ ] **增加射频环境指标。**
  - 在平台允许时采集噪声底、SNR、信道利用率/忙碌时间和每射频链 RSSI。
  - 扫描周边 BSS，分析同信道与邻信道竞争、重叠信道、DFS 状态和拥塞程度。
  - 平台无法提供某项数据时必须降级为 `N/A`，不得推测或伪造。

- [ ] **增加链路层可靠性指标。**
  - 采集 Tx 重传、发送失败、FCS/CRC 错误、Beacon 丢失、重复包等驱动或平台统计。
  - 将重传率和失败率作为干扰、隐藏节点、弱信号与拥塞判断的重要依据。

- [ ] **重构无线速率和 PHY 评分。**
  - 当前速率固定以 6 Mbps 为零分、433 Mbps 为满分且只使用 Tx，对不同频段、制式、
    信道宽度和空间流配置不公平。
  - 同时考虑 Tx/Rx，并优先按“当前协商速率 / 当前配置理论上限”或经验证的设备基线评分。
  - PHY 代际作为能力背景单独展示，避免稳定的旧制式链路因代际本身被重复惩罚。

- [ ] **补齐 Linux 无线数据。**
  - 优先通过 nl80211 等结构化接口获取数据，必要时再解析 `iw` 输出。
  - 补齐 PHY、Wi-Fi 代际、认证、加密、信道宽度、MCS、NSS、重传和信道利用率；为不同
    `iw` 版本、缺失命令、权限不足和驱动字段缺失增加测试与明确提示。

- [ ] **改善无线信息展示入口。**
  - 在适配器详情中展示当前 PHY、频段、信道、RSSI 和 Tx/Rx 速率摘要。
  - 为周边 AP、频谱/信道拥塞和漫游历史提供独立无线分析视图，避免与 LAN ARP 扫描混淆。

## 验收原则

- 未知值、不可用值和真实零值必须在数据模型、界面和评分中严格区分。
- 所有评分维度都应可解释，并显示原始值、阈值/基线及未参与评分的原因。
- Windows 与 Linux 应共享字段语义；平台能力不同可以降级，但不得静默给出误导性结果。
- 新增采集项需要解析单测、评分边界测试和至少一次真实网卡验证；真实验证应覆盖正常、
  弱信号、拥塞、漫游及无线遥测不可用场景。
